File Operations
toc
Requirements
In an internationalized application, users can successfully read/import content from, or write/export content to a file / folder using characters from any script.
The most widely used scripts in the world are: Latin, Cyrillic, Simplified Chinese, Traditional Chinese, Korean, Devanagari, Arabic, and the ones used by the Japanese writing system. File operations involving characters from these scripts provides sufficient internationalization testing coverage.
Typical usage
  • Files containing content created by users
  • User preference files
Typical bugs
  • Corrupted characters in filenames
  • Inability to read content from files with filenames containing non-ASCII characters
How to test
Reading from files and documents
  1. If your product provides the ability to read (i.e. open from and/or import)content from a file or document, rename such input document or file by copying and pasting (if testing product on a non-mobile device) or typing (if testing product on a mobile device) sample strings for each script specified in the section Sample strings below. For typing instructions, read the instructions in the Text input article. If possible, also create and/or rename its parent folder with the same sample string.
  2. Verify whether file or document can be successfully opened without corruption of its content.
  3. Repeat this step for every platform (MacOS, Windows, iOS or Android), that your product is available in.
Writing to files and documents
  1. If your product provides the ability to write (i.e. save to and/or export) content from a file or document, name such output document to use the sample strings for each script specified in the section Sample strings below.
  2. Check whether file or document name matches the sample string, and that it can be re-opened successfully without corruption of its content.
  3. Repeat this step for every platform (MacOS, Windows, or iOS), that your product is available in, and report any issues, as described in the previous section.
